# OpenSpec Workflow

This repository follows a strict OpenSpec flow:
1. Define/maintain stable feature specs in `openspec/specs/`.
2. Track active delivery slices in `openspec/changes/<date>-<slug>/`.
3. Keep implementation checklist in `tasks.md` and verification criteria in `test-plan.md`.
4. Attach command outputs/screenshots as evidence before declaring delivery complete.
